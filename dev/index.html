TGGeometry

TGGeometry.jl is a Julia wrapper around the tg C library for planar geometric predicates. Specifically, it provides:

  • intersects(geom1, geom2)
  • contains(geom1, geom2)
  • touches(geom1, geom2)
  • disjoint(geom1, geom2)
  • equals(geom1, geom2)
  • covers(geom1, geom2)
  • coveredby(geom1, geom2)
  • within(geom1, geom2)

from the DE-9IM model.

It is fully GeoInterface.jl compatible, and is able to accept any combination of GeoInterface-compatible geometries as input (from GeoDataFrames, GeoJSON, ArchGDAL, GeometryOps, and many more packages!).

Quick start

Installation

Install via ]add TGGeometry in the REPL, or Pkg.add("TGGeometry").

Basic usage

Since TGGeometry allows any GeoInterface-compatible geometry as input, let's start with some geometries from NaturalEarth.jl:

using NaturalEarth
using TGGeometry
all_countries = naturalearth("admin_0_countries", 10)
germany = all_countries.geometry[findfirst(==("Germany"), all_countries.NAME)]
belgium = all_countries.geometry[findfirst(==("Belgium"), all_countries.NAME)]
TGGeometry.intersects(germany, belgium)
true
TGGeometry.contains(germany, belgium)
false
TGGeometry.touches(germany, belgium)
true
TGGeometry.disjoint(germany, belgium)
false

You can use any data loader, like GeoJSON, GeoDataFrames, ArchGDAL, etc. You can also construct your own geometries via GeometryBasics, GeoInterface wrapper geometries, or accepted basic types like 2-tuple points.

berlin = (13.4050, 52.5200) # berlin (longitude, latitude)
f # hide`
TGGeometry.contains(germany, berlin)
true

"Preparing" using TGGeom

TGGeometry is fast naturally, but you can make it even faster by "preparing" your geometries by converting them to TGGeoms. This converts the geometries to opaque pointers to a tg geometry - still fully GeoInterface-compatible though, but they have the acceleration benefits and don't have to be continually converted every time you call a predicate.

The way to convert is to call GeoInterface.convert(TGGeometry, geom). This will convert the geometry to a TGGeom and return the new TGGeom object.

Let's see the difference in speed, between using a TGGeom and a GeoJSON polygon:

using Chairmarks, GeoInterface
gj_bench = @be TGGeometry.contains($germany, $berlin) seconds=1
Benchmark: 10947 samples with 1 evaluation
 min    50.524 μs (50 allocs: 49.156 KiB)
 median 78.988 μs (50 allocs: 49.156 KiB)
 mean   82.471 μs (50 allocs: 49.156 KiB, 0.28% gc time)
 max    4.713 ms (50 allocs: 49.156 KiB, 97.78% gc time)
# Convert to TGGeom
germany_tg = GeoInterface.convert(TGGeometry, germany)

tg_bench = @be TGGeometry.contains($germany_tg, $berlin) seconds=1
Benchmark: 30734 samples with 259 evaluations
 min    98.023 ns (1 allocs: 16 bytes)
 median 111.792 ns (1 allocs: 16 bytes)
 mean   117.574 ns (1 allocs: 16 bytes, <0.01% gc time)
 max    70.572 μs (1 allocs: 16 bytes, 99.28% gc time)
faster_factor = Statistics.median(gj_bench).time / Statistics.median(tg_bench).time 
@info "The prepared approach is about $(round(faster_factor, digits=2))x faster!"

Predicates

Predicates from the DE-9IM model are available (but not a full relate function that would return the full DE-9IM matrix).

TGGeometry.intersects — Function
intersects(geom1, geom2)

Check if geom1 and geom2 intersect, as defined by DE-9IM [de9im].

Specifically, this checks that the intersection of two geometries does not result in an empty set.

Arguments

  • geom1: The first geometry. May be any GeoInterface-compatible geometry.
  • geom2: The second geometry. May be any GeoInterface-compatible geometry.
source
TGGeometry.contains — Function
contains(geom1, geom2)

Check if geom1 completely contains geom2, as defined by DE-9IM [de9im].

This means that the boundary and interior of geom2 lie entirely inside geom1, and they cannot intersect the exterior of geom1.

Arguments

  • geom1: The first geometry. May be any GeoInterface-compatible geometry.
  • geom2: The second geometry. May be any GeoInterface-compatible geometry.
source
TGGeometry.touches — Function
touches(geom1, geom2)

Check if geom1 touches geom2, as defined by DE-9IM [de9im].

This means that the intersection of two geometries has at least one point in common, but does not completely contain the other.

Arguments

  • geom1: The first geometry. May be any GeoInterface-compatible geometry.
  • geom2: The second geometry. May be any GeoInterface-compatible geometry.
source
TGGeometry.disjoint — Function
disjoint(geom1, geom2)

Check if geom1 and geom2 are disjoint (completely non-intersecting and separate), as defined by DE-9IM [de9im].

Arguments

TGGeometry.covers — Function
covers(geom1, geom2)

Check if geom1 covers geom2, as defined by DE-9IM [de9im].

This means that no points of geom2 lie outside of geom1.

Arguments

  • geom1: The first geometry. May be any GeoInterface-compatible geometry.
  • geom2: The second geometry. May be any GeoInterface-compatible geometry.
source
TGGeometry.coveredby — Function
coveredby(geom1, geom2)

Check if geom1 is covered by geom2, as defined by DE-9IM [de9im].

This means that no points of geom1 lie outside of geom2.

Equivalent to covers(geom2, geom1) - argument order reversed.

Arguments

  • geom1: The first geometry. May be any GeoInterface-compatible geometry.
  • geom2: The second geometry. May be any GeoInterface-compatible geometry.
source
TGGeometry.within — Function
within(geom1, geom2)

Check if geom1 is within geom2, as defined by DE-9IM [de9im].

This means that the boundary and interior of geom1 lie entirely inside geom2, and they cannot intersect the exterior of geom2.

This is the opposite of contains(geom2, geom1).

Arguments

  • geom1: The first geometry. May be any GeoInterface-compatible geometry.
  • geom2: The second geometry. May be any GeoInterface-compatible geometry.
source

TGGeom

The TGGeom type is a wrapper around the tg_geom object returned by the tg library. It is fully GeoInterface.jl compatible, and is able to accept any combination of GeoInterface-compatible geometries as input (from GeoDataFrames, GeoJSON, ArchGDAL, GeometryOps, and many more packages!).

TGGeometry.TGGeom — Type
TGGeom

A wrapper around a tg_geom pointer. Automatically handles memory management through a finalizer.

source
